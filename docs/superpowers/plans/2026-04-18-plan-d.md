# Plan D — Multi-User Auth + Per-User Favorites

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Google OAuth login (NextAuth v5) so each user has their own favorites, with no auth required to browse the map.

**Architecture:** NextAuth v5 with JWT sessions (no extra DB session tables) + Google OAuth. Favorites table gets a `user_id` column (Google subject ID). The REST API checks the JWT on every favorites request. The map and station data remain fully public. A floating `AuthButton` handles login/logout without touching the map layout.

**Tech Stack:** `next-auth@beta`, Google OAuth 2.0, Neon PostgreSQL, Next.js App Router, `SessionProvider` for client components.

**Out of scope (Plan E):** Public Telegram bot, "en ruta" feature.

## Pre-flight: Google OAuth credentials

Before coding, the developer must:
1. Go to [console.cloud.google.com](https://console.cloud.google.com) → APIs & Services → Credentials → Create OAuth 2.0 Client ID
2. Application type: Web application
3. Authorized redirect URIs: `https://precio-gasolineras-murex.vercel.app/api/auth/callback/google` (and `http://localhost:3000/api/auth/callback/google` for local)
4. Copy **Client ID** and **Client Secret**
5. Set in `.env.local`:
   ```
   AUTH_GOOGLE_ID=<client_id>
   AUTH_GOOGLE_SECRET=<client_secret>
   AUTH_SECRET=<random 32+ char string, generate with: openssl rand -base64 32>
   ```
6. Set the same three vars in Vercel dashboard (Settings → Environment Variables)

```ts
// auth.ts
import NextAuth from 'next-auth';
import Google from 'next-auth/providers/google';

export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [Google],
  session: { strategy: 'jwt' },
  callbacks: {
    jwt({ token, account, profile }) {
      if (account?.providerAccountId) {
        token.userId = `google:${account.providerAccountId}`;
      }
      return token;
    },
    session({ session, token }) {
      if (token.userId) {
        session.user.id = token.userId as string;
      }
      return session;
    },
  },
});
```

## Task 4: Update /api/favorites to require auth

- [ ] **Step 1:** Replace `app/api/favorites/route.ts` entirely:

```ts
// app/api/favorites/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { auth } from '@/auth';
import { getFavorites, addFavorite, removeFavorite } from '@/lib/db';

function unauthorized() {
  return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
}

export async function GET(): Promise<NextResponse> {
  const session = await auth();
  if (!session?.user?.id) return unauthorized();
  const favorites = await getFavorites(session.user.id);
  return NextResponse.json(favorites);
}

export async function POST(req: NextRequest): Promise<NextResponse> {
  const session = await auth();
  if (!session?.user?.id) return unauthorized();
  const { stationId, label } = await req.json();
  if (!stationId) return NextResponse.json({ error: 'stationId required' }, { status: 400 });
  await addFavorite(session.user.id, stationId, label ?? '');
  return NextResponse.json({ ok: true });
}

export async function DELETE(req: NextRequest): Promise<NextResponse> {
  const session = await auth();
  if (!session?.user?.id) return unauthorized();
  const { stationId } = await req.json();
  if (!stationId) return NextResponse.json({ error: 'stationId required' }, { status: 400 });
  await removeFavorite(session.user.id, stationId);
  return NextResponse.json({ ok: true });
}
```
